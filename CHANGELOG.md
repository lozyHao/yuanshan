# 更新日志 / Changelog

本项目的所有重要变更记录于此文件。

## v1.2.0 (2026-06-04)

### 🆕 新增功能

- **参数实时预览**：调整任意参数（边框、颜色、文字、相机参数等）后，预览会自动刷新（带防抖），无需再手动点击「刷新预览」按钮；切换照片、套用模板、导入配置也会自动重绘。
- **预设模板**：侧边栏新增「模板」标签页，内置 6 套常用风格，点击卡片即可一键套用整体风格（基础样式 + 文字模板 + 相机参数），套用后可继续微调：
  - 经典模糊、纯白边框、极简黑框、宽边留白、圆角阴影、底部信息条。
  - 采用两列卡片布局，缩略图依据预设真实配置（背景 / 圆角 / 文字条颜色）动态渲染，效果一目了然。
- **配置导入 / 导出**：可将「基础配置 + 文字模板（含中线位置）+ 相机参数」整合导出为单个 JSON 文件，并支持导入还原，方便备份、换机迁移与分享。导出文件与内置预设模板结构同构，你也可以把调好的配置当成自定义模板分享。
- **安装路径可选**：Windows 安装包（NSIS）支持在安装向导中自定义安装目录。

### ⚡ 优化与重构

- **画板笔迹平滑**：手绘画板改用「中点二次贝塞尔」算法 + 恒定笔宽 + 圆角线帽，解决快速绘制时笔迹断点不连续的问题；鼠标坐标按画布缩放换算避免错位；单击即可落点。
- **内存优化**：修复多处 `objectURL` 未释放导致的内存泄漏（预览图、水印、缩略图用完即释放；移除文件时统一回收）；Worker 线程用完即终止，回调按完成进度清理，避免堆积。
- **代码瘦身**：删除无用的 `outputWorker.ts` 死代码；抽离绘制管线、图片加载、默认配置工厂（`createDefaultBasic`）与配置读写组合式（`useConfigIO`），减少重复、统一逻辑。
- **临时文件清理**：应用启动时自动清理上次残留的临时文件，避免磁盘占用无限增长。
- **开发体验**：为 Pinia store 增加热更新支持（`acceptHMRUpdate`）；清理生产环境遗留的调试日志。

### 🐛 Bug 修复

- **EXIF 缺失不再绘制**：修复照片缺少某项 EXIF 时，会把 `ISOnull`、`undefined`、`NaN` 等占位文字绘制到成图上的问题（从取值源头拦截空值）。
- **EXIF 读取竞态**：修复元数据异步加载未完成就被读取，可能导致参数为空的问题。
- **输出格式**：修复输出 MIME 类型 `image/jpg` → `image/jpeg`，避免部分环境回退为 PNG。
- **错误提示**：修复文件保存 / 复制失败被静默吞掉的问题，改为返回真实错误信息。
- **配置导入报错**：修复修改 store 后热更新残留旧实例导致的 `setAll is not a function`。

### 🔒 安全

- **修复任意文件读取漏洞**：自定义协议 `local-yuanshan` 增加路径白名单校验，仅允许读取应用临时目录内的文件，阻断路径穿越。
- **恢复安全策略**：重新开启 `webSecurity`，移除 `bypassCSP` 与对 Electron 安全告警的屏蔽。
- 移除将文件二进制内容打印到控制台的日志。
